# Releasing modular-open-code-review

Internal maintainer notes. Not required for consumers.

## `config/release.json`

Tracks **`consumer_ref`**: the git tag or branch downstream repos should pin in reusable workflow `uses:` lines (e.g. `v1.0.0`).

Do not edit this file manually to cut a release. The release workflow updates it after a tag is published.

## Release workflow

[`.github/workflows/release.yml`](../.github/workflows/release.yml):

1. **On tag push (`v*`)** — creates a GitHub Release if one does not exist yet, then commits updated `@ref` pins on `main`.
2. **On workflow_dispatch** — creates the tag (if missing on origin), creates the Release, and commits the ref bump.

```bash
# Option A: push a tag
git tag v1.0.0 && git push origin v1.0.0

# Option B: dispatch from GitHub Actions UI with version 1.0.0
```

After a successful run, [`scripts/bump-consumer-refs.sh`](../scripts/bump-consumer-refs.sh) updates `README.md`, `examples/*.yml`, and `config/release.json` on `main` with a follow-up commit like `chore: bump consumer refs to v1.0.0`. There is no separate release branch.
